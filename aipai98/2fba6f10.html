人力资源系统配置

步骤 安装 CentOS

创建虚拟机

在“选择虚拟机硬件兼容性”界面，使用默认设置即可。

在“安装客户机操作系统”界面，选择“稍后安装操作系统”单选按钮。

在“选择客户机操作系统”界面，选择Linux和Centos7 64位

在“命名虚拟机”界面，设置虚拟机名称及存放位置。

在“处理器配置”界面，配置处理器数量及每个处理器的内核数量。

在“此虚拟机的内存”界面，配置内存大小。

在“网络类型”界面，配置网络。

在“选择 I/O 控制器类型”界面，选择默认设置即可。

在“选择磁盘类型”界面，选择默认设置即可。

在“选择磁盘”界面，选择“创建新虚拟磁盘”单选按钮

在“指定磁盘容量”界面，设置磁盘大小。在“指定磁盘文件”界面，选择默认设置即可。

在“指定磁盘文件”界面，选择默认设置即可。

在“已准备好创建虚拟机”界面，单击“完成”按钮开始创建虚拟机。

虚拟机创建完成后，双击虚拟机设备中的 CD/DVD（IDE），弹出“虚拟机设置”对话框，CD/DVD（IDE）设备“连接”设置中，选择“使用 ISO 映像文件”单选按钮，然后选择之前下载的 CentOS 7 光盘映像文件，单击“确定”按钮。

CD/DVD（IDE）设置修改完成后，虚拟机创建完成。

2．安装 CentOS 7

单击创建的虚拟机的“开启此虚拟机”按钮，启动虚拟机。

虚拟机启动后，进入 CentOS 7 安装界面，单击进入安装界面，通过上下键，使光标选中第一项 Install CentOS 7，然后按【Enter】键，开始安装 CentOS 7。

CentOS 7 安装中。

在 WELCOME TO CENTOS 7 界面设置语言，选择 English，单击 Continue 按钮。

在 INSTALLATION SUMMARY 界面安装总览，在这里可以完成 CentOS 7 版本 Linux 的全部设置。

在 DATE & TIME 界面设置时区，选择 Asia-Shanghai。

在 KEYBOARD LAYOUT 界面设置键盘，使用默认设置 English（US）即可。

在 LANGUAGE SUPPORT 界面设置语言支持，可以使用默认的 English，也可以自行添加简体中文的支持。

在 INSTALLATION SOURCE 界面安装资源，使用默认设置 Local media 本地媒体文件即可。

在 SOFTWARE SELECTION 界面安装选择软件，字符界面安装选择 Minimal Install 或者 Basic Web Server。图形界面安装选择 Server with GUI 或者 GNOME Desktop。字符界面与图形界面安装过程相同，只在这一步有区分。

在 INSTALLATION DESTINATION 界面设置安装位置，即进行系统分区。选中在创建虚拟机时设置的 30 GB 虚拟硬盘。

滚动鼠标，在页面下方找到 Other Storage Options→Partitioning，选中 I will configure partitioning 自定义分区，然后单击 Done 按钮。

选择 Standard Partition 标准分区，单击页面左下角的“+”按钮添加分区。

/boot 分区设置为 1 GB。

swap 分区设置为 4 GB。

/根分区设置为 25 GB。

分区完成后单击 Done 按钮，弹出 SUMMARY OF CHANGES 对话框，单击 Accept Changes 按钮。

KDUMP 使用默认设置即可。

在 NETWORK & HOST NAME 界面设置网络和主机名，在 Host name 文本框中输入主机名，单击 Apply 按钮即可设置主机名。

完成所有设置后，单击 Begin Installation 按钮，开始安装 CentOS 7。

在 USER SETTINGS 界面进行用户设置，这里需要设置 root 用户的密码。

在 ROOT PASSWORD 界面设置 root 用户的密码。

在 CREATE USER 界面创建用户，可以不进行创建，安装完成后 root 用户也可以创建新的用户。

CentOS 7 安装完成之后，单击 Reboot 按钮重启系统。

CentOS 7 系统启动。

系统启动成功，进入用户登录界面。

输入 root 用户的用户名和刚才设置的密码，登录系统。 密码输入时不显示

root 用户登录成功。

步骤 2：安装配置 JDK

步骤 3：安装配置 MySQL

人力资源系统数据库配置新建人力资源系统数据库 suthr。

进入 suthr 数据库

使用 suthr.sql 命令初始化数据库。

数据库初始化成功，查看数据库表。

步骤 4：安装配置 Tomcat

1．安装 Tomcat 8

将 Tomcat 8 安装文件上传到 Linux 对应的文件夹中。这里使用 WinSCP 软件上传到/home 文件夹下。

使用 tar -zxvf 解压安装文件。

解压完成后，在 home 目录下多出一个 apache-tomcat-8.5.63 文件夹。

将 apache-tomcat-8.5.63 文件重命名为 tomcat。

出现 Tomcat started，表明 Tomcat 启动成功。

启动 Tomcat 后，在浏览器访问“http://IP 地址:8080”，出现 Tomcat 主页，则表明 Tomcat 启动成功。

若用主机访问虚拟机，需关闭虚拟机防火墙。

查看防火墙运行状态的命令。

停止防火墙运行的命令。

永久关闭防火墙的命令。

2．配置 Tomcat 8 在 tomcat8\bin\目录中，编辑 catalina.sh 文件。

在 cygwin=false 前，加入 JAVA_OPTS=”-Xms512m -Xmx512m”。

步骤 5：WebAPP 部署将 suthr.war 包复制到 Tomcat 安装目录下的 webapps 目录。

在 webapps 目录下，会自动生成一个 suthr 目录，若没有自动生成，可重启 Tomcat。

进入 suthr 目录下的数据库配置目录。

/home/tomcat/webapps/suthr/WEB-INF/classes/db/jdbc：

在数据库配置目录下的 jdbc.properties 文件中，数据库的用户名密码要和之前配置的数据库的用户名密码一致。

若修改了数据库配置文件，需重启 Tomcat。建议尽量不要修改配置文件内容。

步骤 6：访问人力资源综合服务系统通过浏览器访问“http://IP 地址:8080/suthr”，出现人力资源系统登录页面，表明人力资源系统测试环境搭建成功。
